# Урок 4. Промпт триажа: роль, таксономия, правила, схема

Промпт триажа — не «просьба к чат-боту», а рабочий документ из четырёх
обязательных частей. Полный текст лежит в
[`analyzer/prompt.md`](../analyzer/prompt.md) — он специально вынесен в файл
репозитория, а не зашит в код и не живёт в чьём-то чате.

## Четыре части промпта

### 1 · Роль
«Ты — дежурный QA-lead, утро после ночного прогона». Роль задаёт режим:
диагноз, а не болтовня. Модель не рассуждает вслух — ставит вердикты.

### 2 · Таксономия с признаками
Пять категорий из урока 2 — с признаками каждой. Модель **выбирает из закрытого
списка**, а не изобретает свои категории.

### 3 · Правила
Самая ценная часть — накопленные правила разбора. Ключевые:

```
1. Вердикт без улик не принимается: в evidence — конкретные строки
   из трейсбека, кода теста или контракта.
2. Не выдумывай. Мало фактов — needs_human: true и confidence ниже 0.6.
3. Различай failure и error: assert не сошёлся — решай, кто нарушил
   контракт; исключение до assert'а — смотри в обвязку и инфраструктуру.
```

Плюс правила, выросшие из реальных ошибок разбора: «категорию код_автотеста vs
инструментарий определяет МЕСТО ошибки», «клиентский таймаут — всегда спорный»,
«одинаковые падения обязаны получить одинаковую категорию». Так промпт
становится **базой знаний команды о том, как разбирать падения**.

### 4 · Схема ответа (JSON)
Модель обязана ответить структурой, которую можно распарсить кодом:

```json
{
  "test": "tests.test_practice_contracts::test_temperature_conversion",
  "category": "продукт",
  "confidence": 1.0,
  "root_cause": "GET /practice/temperature считает F по неверной формуле: при C=100 вернул 180.0 вместо 212.",
  "evidence": "Контракт: F = C × 9/5 + 32 → 212. Ответ сервиса: {'fahrenheit': 180.0}. Тест утверждает ровно контракт.",
  "suggested_action": "Завести баг на продукт; тест не менять.",
  "needs_human": false,
  "draft_bug_title": "GET /practice/temperature: неверная формула конвертации при celsius=100"
}
```

Обратите внимание на три поля:

- `category` — только из enum: модель физически не может вернуть категорию
  «Super Bug» (схема в [`analyzer/schema.py`](../analyzer/schema.py) её отбракует);
- `evidence` — **обязательное** поле, пустая строка не проходит валидацию;
- `needs_human` — легальный способ сказать «не знаю».

## Главное правило: вердикт без улик не принимается

В `evidence` должны быть конкретные строки из трейсбека, кода или контракта.
Нет улик — вердикт на выброс, **какой бы ни была confidence**.

| ✗ Мнение | ✓ Улика |
|---|---|
| «Похоже на баг продукта» | «Контракт: 422 при amount > from_balance; сервис вернул 200 и new_balance = −4900» |

Почему это так важно: улики — это **трассируемость вердикта**. Человек, читающий
сводку триажа, должен иметь возможность проверить каждый диагноз, не
перечитывая логи с нуля. Улика — это ссылка «проверь здесь».

И вторая причина: `confidence` — это **самооценка модели, а не истина**.
Значения плавают от запуска к запуску, а высокая уверенность прекрасно
сочетается с выдуманной уликой (вспомните урок 3). Проверяемая улика — да,
цифра уверенности — нет.

Мини-задача. Модель выдала «продукт, confidence 0.95», но в `evidence` — общие
слова без цитат. Ваши действия?

<details>
<summary>Ответ</summary>

Отклонить вердикт: без улик не принимается — какой бы высокой ни была
confidence. Не принять, не «поднять temperature и перегенерировать», не завести
баг. Разобрать вручную или дослать модели недостающий контекст.
</details>

## Почему промпт — файл в репозитории

- **Diff в ревью.** Изменение правила разбора проходит code review, как код.
- **Версия в истории.** Понятно, каким промптом был сделан вчерашний триаж.
- **Общий уровень качества.** Вся команда разбирает падения по одним правилам,
  а не «кто как привык».

## Уровень 0: попробовать без кода и ключей

Весь этот подход работает даже вручную, в любом чате с LLM:

1. Откройте промпт [`analyzer/prompt.md`](../analyzer/prompt.md) и эталонный
   отчёт [`reports/sample_report.xml`](../reports/sample_report.xml).
2. Скопируйте в чат промпт + одно падение из отчёта.
3. Получите структурированный вердикт. Проверьте улики. Сравните с
   [`examples/triage.md`](../examples/triage.md).

Это «уровень 0» — в нём нет автоматизации, но есть вся суть: роль, таксономия,
правила, схема, улики.

## ✅ Как надо / ❌ как не надо

| ❌ Как не надо | ✅ Как надо |
|---|---|
| «Посмотри, почему упали тесты» в свободной форме | Роль + таксономия + правила + схема ответа |
| Модель сама придумывает категории | Закрытый enum, валидация схемы кодом |
| Принимать вердикт из-за высокой confidence | Принимать вердикт из-за проверяемых улик |
| Промпт живёт в чьём-то чате | Промпт — версионируемый файл в репозитории |
| Наказывать «не знаю» и требовать всегда ответ | needs_human — нормальный исход при нехватке фактов |
| Просить и диагноз, и эссе о качестве проекта | Один вызов — одна задача: диагноз по схеме |
