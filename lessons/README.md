# 📚 Уроки: как анализировать результаты прогонов вместе с ИИ

Мини-учебник для тестировщиков о **триаже падений автотестов** — разборе красного
прогона с помощью LLM. Здесь теория и принципы, а не инструкция к конкретному
инструменту: анализатор из этого репозитория используется как живой пример
**минимального конвейера**, который вы можете повторить и перекроить под свой проект.

Практиковаться можно на публичном тренажёре
[BookShelf API Trainer](https://qahacking.up.railway.app): в разделе `/practice`
контракты честно задокументированы в [спеке](https://qahacking.up.railway.app/docs),
а баги намеренно спрятаны в реализации — поэтому части тестов из `tests/`
*положено* падать, и всегда есть что триажить.

## Оглавление

| Урок | О чём | Главная мысль |
|------|-------|---------------|
| [Урок 1. Зачем нужен триаж](lesson-01-why-triage.md) | Экономика разбора красного прогона | Отчёт отвечает «что упало», триаж — «почему и что делать» |
| [Урок 2. Таксономия вердиктов](lesson-02-taxonomy.md) | Пять категорий причин + честное «не знаю» | Диагноз ставится из закрытого списка, у каждой категории — своё действие |
| [Урок 3. Пакет контекста](lesson-03-context-package.md) | Что должна видеть модель, чтобы не гадать | ~70% качества вердикта решает вход, а не модель |
| [Урок 4. Промпт триажа](lesson-04-triage-prompt.md) | Роль, правила, схема ответа | Вердикт без улик не принимается |
| [Урок 5. Конвейер](lesson-05-pipeline.md) | Минимальный пайплайн триажа | Код — для гарантий, модель — только для смысла |
| [Урок 6. Баги, метрики, границы](lesson-06-bugs-metrics-safety.md) | Автозаведение, human-in-the-loop, безопасность | ИИ готовит черновик — решение принимает человек |
| [Урок 7. Свой триаж](lesson-07-make-it-yours.md) | Что крутить под свой проект и куда расти | Меняются вход и правила — ядро конвейера то же |

## Как читать

- Уроки короткие и идут по порядку: от «зачем» к «как» и «куда дальше».
- В каждом уроке есть блок **«Как надо / как не надо»** — это и есть суть.
- Примеры кода и файлов — из этого репозитория: промпт
  [`analyzer/prompt.md`](../analyzer/prompt.md), схема
  [`analyzer/schema.py`](../analyzer/schema.py), конвейер
  [`analyzer/analyzer.py`](../analyzer/analyzer.py).
- Ничего запускать не обязательно: эталонный отчёт лежит в
  [`reports/sample_report.xml`](../reports/sample_report.xml), готовый результат
  триажа — в [`examples/triage.md`](../examples/triage.md).
