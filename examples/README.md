# Эталонный прогон (для показа без ключа)

Это **реальный вывод** инструмента на живом API `BookShelf` (модель
`deepseek-v4-flash`), с задеплоенными эндпоинтами челленджера. Файлы закоммичены,
чтобы на занятии можно было открыть готовый результат на экране, не запуская
триаж и не завися от ключа/сети.

- [`triage.html`](triage.html) — красивый отчёт для браузера (открыть напрямую);
- [`triage.md`](triage.md) — та же сводка в markdown;
- [`verdicts.json`](verdicts.json) — вердикты по каждому падению для машин;
- [`issues.json`](issues.json) — черновики багов.

## Картина прогона

**11 passed / 10 failed / 4 error** → 14 красных → **10 причин** после группировки,
все пять категорий:

| Категория | Причин | Примеры |
|-----------|--------|---------|
| продукт | 3 | temperature (180≠212), transfer (уходит в минус), утечка приватных полей |
| код_автотеста | 4 | `name` vs `title`, `price=-10`, GET списка ждёт 201, спорный таймаут (⚠ needs_human) |
| инфраструктура | 1 | genres: ConnectionError на недоступный хост |
| инструментарий | 1 | фикстура `reader` ждёт 200, а `/auth/token` отдаёт 201 — **4 теста слиплись в один инцидент** |
| flaky | 1 | `/practice/unstable` отдаёт 500 в ~30% прогонов |

> 25 тестов покрывают методы (GET/POST/PATCH/DELETE/HEAD/OPTIONS) и жизненные циклы
> ресурсов. Четыре reader-падения `triage.md` схлопывает в один инцидент. Спорные
> кейсы помечены `needs_human` и НЕ попадают в черновики багов.

Чтобы получить такой же вывод самому:
`python -m pytest -q --junitxml=reports/report.xml` → `python analyzer/analyzer.py reports/report.xml`
(нужен ключ модели в `.env`).
